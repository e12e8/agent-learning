# Agent 开发学习计划（分阶段 + 可验收里程碑）

下面是经过优化的学习与交付计划，强调可验证的产出与求职导向的里程碑。

## 核心原则（必读）
- 必做 vs 可选：每日任务分为“必做（核心）”和“加分（可选）”。必做项固定必须完成；加分项视进度而定。
- 验收标准：每周给出可运行的产出（README、示例、测试、CI 或 PR）。

## 第一阶段：考试期间（明天-14号，每天约3小时）

### 目标：夯实基础，提升代码质量并产出可展示的小改进

**每日安排（建议，固定产出）**：
- 必做：45 分钟理论 + 90 分钟代码实践 + 45 分钟总结/复盘（每天产出一条 commit/PR 或一段 README 更新）
- 加分：开源阅读或额外实验（完成则记录在复盘中）

**必做清单（验收项）**：
1. 类型注解与 Pydantic：覆盖主要模块（PR + 类型检查通过）
2. 单元测试：将关键模块覆盖率提升至 >= 60%（添加测试并在本地通过）
3. 异常处理与日志：添加结构化日志与基础错误边界
4. CI 初步：配置 GitHub Actions（至少有一个测试/检查工作流）

**短期成果**：每天一个小 PR / README 更新；阶段结束时有一份变更记录与测试报告。

## 第二阶段：假期强化（15号开始，每天约8小时）

划分为四个按周的里程碑，每周有清晰的验收条件。

### 第1周（模块化与接口设计）
- 目标：定义模块边界、插件接口与示例插件。
- 验收：设计文档（README）、示例插件和交互示例脚本。

### 第2周（状态管理与持久化）
- 目标：实现持久化状态管理（文件或轻量 DB）、实现恢复能力。
- 验收：持久化示例、自动化测试、说明文档。

### 第3周（多 Agent 协作与工具组合）
- 目标：实现 Agent 间通信、任务分配与工具动态组合。
- 验收：协作示例脚本、多 Agent 集成测试、性能基线记录。

### 第4周（集成、性能与安全）
- 目标：整合前面工作，进行异步优化、缓存、基本权限控制与监控。
- 验收：可运行的 demo（Docker Compose 或部署脚本）、性能报告、基本安全策略说明。

## 第三阶段：项目实战（项目周期视选择而定）

### 目标：产出可展示的个人代表作（MVP + 扩展）

推荐流程：
- Week 1–2：MVP（2 周内交付）——聚焦核心功能（例如：文档摘要 + 问答）。
- Week 3–4：扩展（加入 OCR、翻译或代码分析等高级特性）。

**产出要求**：
- 项目 README（功能、设计、运行示例）
- 部署说明（Docker Compose 或一键启动脚本）
- 演示素材（GIF 或 1 分钟视频）
- 面试素材：一页“项目亮点 + 技术难点 + 解决方案”文档

## 面试与求职准备（新增）
- 每周准备一页项目亮点，用于简历与面试展示。
- 准备 5 个 STAR 技术回答（示例：系统设计、异常处理、性能优化、多 Agent 协作、测试策略）。
- 在 GitHub 保持持续贡献（建议每 2–3 天一次 commit/PR），建立可被追溯的活动记录。

## 技能清单（学习 + 必须产出）
- Python 异步（async/await）：写一个简单的异步任务处理示例
- FastAPI：实现并部署一个 1 个端点的 demo
- LangChain / Agent 框架：完成一个最小 Agent 工作流示例
- Pydantic：类型和配置管理示例
- Docker & CI：Dockerfile + GitHub Actions 工作流示例

## 质量、监控与安全
- 日志与 metrics：集成基础指标（Prometheus 或内置 endpoint），并在 README 说明查看方式。
- 输入验证与最小权限：在设计文档中写明策略并提供示例实现。

## 时间与期望值调整
- 建议表述："40 天达成可面试的项目成果"（比“企业用人标准”更可验证且更现实）。

## 学习反馈与协作
- 每周至少一次 code review（社区或同学），并将收到的 3 条改进意见记录在复盘中。
- 每周产出一份复盘（含三条待改进项）以保证快速闭环。

## 优先级行动计划（快速落地）
1. 两周内做出 MVP 并推到 GitHub（优先级最高）。
2. 下一周完善测试 + CI + 文档，保证可 demo. 
3. 随后两周优化架构与加入监控/性能测试。
